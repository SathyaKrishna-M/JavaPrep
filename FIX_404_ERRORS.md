# Fix 404 Errors - Quick Guide

## Problem
You're seeing 404 errors for:
- `layout.css`
- `main-app.js`
- `app-pages-internals.js`

## Solution

### Step 1: Stop Any Running Servers
1. Press `Ctrl+C` in any terminal windows running `npm run dev`
2. Close any browser tabs with the app open

### Step 2: Clean Build Cache
Run these commands in PowerShell:

```powershell
# Remove build cache
Remove-Item -Path .next -Recurse -Force -ErrorAction SilentlyContinue
Remove-Item -Path node_modules\.cache -Recurse -Force -ErrorAction SilentlyContinue
```

Or use the quick start script:
```powershell
.\start-dev.ps1
```

### Step 3: Start Development Server
```powershell
npm run dev
```

**Wait for the server to start!** You should see:
```
✓ Ready in X seconds
○ Local: http://localhost:3000
```

### Step 4: Clear Browser Cache
1. **Hard Refresh**: Press `Ctrl + F5` (or `Ctrl + Shift + R`)
2. **Or Clear Cache**:
   - Press `Ctrl + Shift + Delete`
   - Select "Cached images and files"
   - Click "Clear data"

### Step 5: Open the App
1. Go to `http://localhost:3000`
2. Make sure the URL is exactly `http://localhost:3000` (not just `localhost:3000`)
3. Wait for the page to load completely

## Why This Happens

These errors occur when:
1. **Development server is not running** - The files are generated by the dev server
2. **Browser cached old file references** - Old build artifacts are cached
3. **Production build files** - You might be looking at production build files instead of dev server
4. **Server crashed** - The dev server stopped unexpectedly

## Verification

### Check if Server is Running
```powershell
Get-NetTCPConnection -LocalPort 3000 -ErrorAction SilentlyContinue
```

If this returns nothing, the server is not running.

### Check Terminal Output
Look for these messages in your terminal:
- ✅ `✓ Ready in X seconds` - Server is ready
- ✅ `○ Local: http://localhost:3000` - Server is running
- ❌ Any error messages - Fix those first

## Common Issues

### Issue 1: Port 3000 Already in Use
**Solution**: Next.js will automatically try the next available port (3001, 3002, etc.)
- Check the terminal output for the actual port number
- Use that port in your browser

### Issue 2: TypeScript Errors
**Solution**: Fix any TypeScript errors first
```powershell
npm run build
```
This will show you any compilation errors.

### Issue 3: Missing Dependencies
**Solution**: Reinstall dependencies
```powershell
Remove-Item -Path node_modules -Recurse -Force
Remove-Item -Path package-lock.json -Force
npm install
```

### Issue 4: Browser Still Shows Errors
**Solution**: 
1. Close all browser tabs
2. Clear browser cache completely
3. Restart browser
4. Open `http://localhost:3000` in a new tab

## Quick Fix Script

Use the provided `start-dev.ps1` script:
```powershell
.\start-dev.ps1
```

This script will:
1. Install dependencies if needed
2. Clean build cache
3. Start the development server

## Still Having Issues?

1. **Check Node.js version**:
   ```powershell
   node --version
   ```
   Should be Node.js 18 or higher.

2. **Check if Next.js is installed**:
   ```powershell
   npm list next
   ```

3. **Try a fresh install**:
   ```powershell
   Remove-Item -Path node_modules -Recurse -Force
   Remove-Item -Path package-lock.json -Force
   Remove-Item -Path .next -Recurse -Force
   npm install
   npm run dev
   ```

4. **Check for error messages** in the terminal when starting the server

## Expected Behavior

When everything is working correctly:
- ✅ Server starts without errors
- ✅ Terminal shows "Ready" message
- ✅ Browser loads `http://localhost:3000` without 404 errors
- ✅ Page displays correctly
- ✅ No errors in browser console (F12)
